# Strands Agent Daily Report, 2026-04-21

## A) What was built today

Added explicit launch-time configuration controls for the TUI prototype:
- `strands-agent --runtime {fake,live}`
- `strands-agent --model <model-id>`

Also updated the app status line so each run now shows the active runtime label, mode, selected model, and turn count in one place.

## B) Why this step matters for understanding Strands

This makes the runtime seam more legible. Steve can now switch between the deterministic fake runtime and the real Strands-backed live runtime intentionally, without relying only on environment variables. That makes it easier to study what is Strands behavior versus what is local harness behavior.

## C) What Steve will learn from it

- how to keep a TUI decoupled from the underlying Strands runtime
- how runtime/provider selection can be surfaced as a first-class product concern
- how a fake runtime can coexist with a live runtime without muddying the app boundary
- how launch-time config helps compare learning environments quickly

## D) How to run it

```bash
cd /home/steve/.openclaw/workspace/strands_agent
source .venv/bin/activate
strands-agent
```

Override runtime/model for a single launch:

```bash
strands-agent --runtime fake
strands-agent --runtime live --model gpt-4.1-mini
```

Run tests:

```bash
pytest
```

Verify CLI wiring:

```bash
strands-agent --help
```
